# Shock Q&A Challenge

## Game Introduction
The Q&A game is a quiz gameplay that supports external question import. Players must answer questions within a set time limit. A correct answer leads to the next question, while an incorrect answer triggers a shock punishment, and the question is placed back in the queue to be answered later.

## Game Features
+ 📚 **Customizable Question Bank**: Supports importing external question files in JSON format
+ ⚡ **Shock Punishment**: Incorrect answers trigger the shock device for punishment
+ ⏰ **Time Limit**: Each question has a time limit; exceeding it is considered an incorrect answer
+ 🎮 **Multiple Control Methods**: Supports mouse clicking and QTZ keypad device operation
+ 🔒 **Manual Start Mode**: Optional manual start mode, activated via the automatic lock device key

## Equipment Requirements
### Required Equipment

## Game Parameter Configuration
### Question Settings
+ **Question File Path**: Specifies the path to the JSON file containing the questions
    - Default: `<OUTTER_GAME>/QA-game/女仆行为规范考核题库.json`
    - Supports the `<OUTTER_GAME>` tag pointing to the packaged outter-game folder

### Punishment Settings
+ **Shock Intensity**: Shock intensity for incorrect answers (10-100)
+ **Shock Duration**: Duration of the shock (1-10 seconds)

### Time Settings
+ **Answer Time Limit**: Time limit for answering each question (10-300 seconds, 0 means no limit)

### Start Settings
+ **Manual Start**: When enabled, the game starts only after pressing the automatic lock device key

## Question File Format
The question file should be in JSON format with the following structure:

```json
{
  "questions": [
    {
      "question": "Question text",
      "options": ["Option A", "Option B", "Option C", "Option D"],
      "correct": 0,
      "explanation": "Answer explanation (optional)"
    }
  ]
}
```

### Field Descriptions
+ `question`: Question text
+ `options`: Array of options
+ `correct`: Index of the correct answer (starting from 0)
+ `explanation`: Answer explanation (optional)

## Game Flow
1. **Game Preparation**
    - Load the question file
    - Check device connection status
    - Initialize game parameters
2. **Game Start**
    - Manual start mode: Wait for the automatic lock device key signal
    - Auto-start mode: Immediately begin with the first question
3. **Answering Process**
    - Display the question and options
    - Automatic lock device activates (prevents mid-game exit)
    - Countdown begins
    - Wait for the player to select an answer
4. **Answer Judgment**
    - **Correct**: Display correct prompt, proceed to the next question
    - **Incorrect or Timeout**: Trigger shock punishment, question is moved to the end of the queue to be answered again
5. **Game End**
    - Game ends when all questions are answered correctly
    - Automatic lock device deactivates
    - Display game statistics

## Control Methods
### Mouse Control
+ Click the option button to select an answer

### QTZ Keypad Control
+ **Key 0**: Move selection up
+ **Key 1**: Move selection down / Confirm selection
+ The selected option is highlighted

## Important Notes
1. Ensure the shock device is correctly connected and configured. Please adjust the voltage intensity yourself. You are responsible for any shock injuries caused by excessive intensity.
2. The automatic lock device may be unstable due to network conditions and other factors. Always have escape tools prepared.
